# 设置页重构与批量操作计划

状态：第一版已完成。设置中心、数据备份页、批量图片下载/删除、批量对话删除和输入确认弹窗均已落地。

## 背景

当前设置弹窗的标题是“接口设置”，但内容已经包含接口配置和数据备份。随着后续加入批量下载、批量删除图片、批量删除对话等能力，继续把所有控件堆在同一个页面里会让设置区变得拥挤，也会让图片库承担过多管理功能。

本次重构目标是把设置弹窗升级为“设置中心”，用左侧导航承载不同设置分类，把低频但重要的数据管理操作统一放在设置里，减少图片库主界面的控件压力。

## 导航结构

设置弹窗采用左右布局。

左侧导航：

- 接口
- 数据备份
- 批量操作

右侧根据当前导航展示对应内容。

移动端可以退化为顶部横向标签，或保持简化后的纵向导航；第一版优先保证可用和信息不拥挤。

## 页面规划

### 接口

保留当前接口配置能力：

- OpenAI API key
- API Base URL
- API key 本地保存提示
- “没有 API Key？”链接

整个弹窗标题改为“设置”，右侧内容标题使用“接口”或“接口设置”。

### 数据备份

保留当前完整备份恢复能力：

- 导出备份
- 恢复备份
- 说明：导出会话、消息和图片；API key 不会写入备份。

恢复备份继续使用覆盖当前本地数据的策略，并保留二次确认。

### 批量操作

批量操作页用于管理本地数据中的批处理任务。第一版包含两个区块：图片、对话。

#### 图片

能力：

- 展示图片总数。
- 展示图片列表，支持多选。
- 全选、清空选择。
- 下载选中图片。
- 删除选中图片。

图片列表建议显示：

- 缩略图。
- 图片名称。
- 来源：生成图 / 导入图。
- 创建时间。
- 文件大小或尺寸，可择一显示，避免列表过密。

下载策略：

- 复用现有 ZIP 下载能力。
- 将选中的图片 Blob 打包为 ZIP。
- 文件名沿用当前图片库下载命名规则。

删除策略：

- 删除选中图片的 `imageAssets`。
- 删除对应 `imageBlobs`。
- 从当前输入框引用中移除这些图片。
- 历史消息不删除，继续显示“图片已删除”的占位。
- 删除后刷新图片库和本地存储空间统计。

删除确认：

- 弹出输入确认弹窗。
- 用户必须输入：

```text
我确认删除选中的 N 个图片
```

- 输入完全匹配后，确认删除按钮才可用。

#### 对话

能力：

- 展示对话总数。
- 展示对话列表，支持多选。
- 全选、清空选择。
- 删除选中对话。

对话列表建议显示：

- 对话标题。
- 摘要或更新时间。

删除策略：

- 删除选中 conversations。
- 删除这些 conversation 下的 messages。
- 图片库中的图片保留。
- 如果当前会话被删除，切换到剩余第一个会话。
- 清空输入框草稿和当前引用图。

删除确认：

- 弹出输入确认弹窗。
- 用户必须输入：

```text
我确认删除选中的 N 个对话
```

- 输入完全匹配后，确认删除按钮才可用。

## 通用确认弹窗

新增可复用的输入确认弹窗，用于高风险操作。

建议 props：

- `title`
- `description`
- `confirmText`
- `confirmLabel`

交互：

- 显示需要输入的确认文本。
- 输入框实时匹配。
- 只有完全匹配时，确认按钮可用。
- 支持取消。

后续也可以复用到“恢复备份覆盖当前数据”等高风险操作。

## 第一版取舍

第一版不做复杂搜索、筛选和排序，避免把设置页做成第二套图片库或会话库。

优先完成：

1. 设置弹窗左右布局重构。
2. 接口、数据备份、批量操作三个页面。
3. 图片批量下载。
4. 图片批量删除。
5. 对话批量删除。
6. 输入确认弹窗。

后续如果本地数据规模变大，再考虑为批量操作页增加搜索和筛选。
